# Changelog

## v3.0.6

- `QzoneSession` 新增从机器人客户端重新抓取最新 QQ 空间 Cookie 的恢复路径
- 最近动态流重试不再只是清空 session，而是优先刷新 Cookie 后重登
- 底层 HTTP 客户端在明确登录失效时，会优先尝试刷新 Cookie 再重试请求

## v3.0.5

- 自动评论查询动态流时，新增最近动态接口的自动重登重试
- 对“动态接口空响应”“动态流无权限”“登录态短暂失效”“响应格式异常”改为软跳过本轮任务，不再每轮都打印整段异常栈
- 用于缓解 QQ 空间动态流接口偶发抖动时的噪声日志和误判失败

## v3.0.4

- 在自动评论调度器中改为使用专用的评论偏移配置，而不是复用发布偏移配置
- 确保自动评论过滤能够正确检测已有的自身评论，同时结合内存帖子和数据库记录，避免对同一条动态重复评论
- 自动评论任务对每条帖子增加局部错误处理，单条失败不会中断本轮剩余帖子

## v3.0.3

- 重构自动发说说、自动评论的随机偏移调度逻辑，围绕 cron 基准时间做正负偏移

## v3.0.2

- 修复 QZone API 返回空字符串时的 JSON 解析异常
- 新增接口响应兜底解析，避免空响应和非法 JSON 直接导致插件中断
- 修复 403 被误判为登录失效的问题，仅在明确登录失效时才重登
- 统一接口错误常量和错误映射，细化“无权限查看”“登录失效”“响应格式异常”等提示
